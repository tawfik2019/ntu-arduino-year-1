Using Arduino

Defining your pins

You should define the pins you use are using as constants all together at the top of your code.

Example

Using const:

const int REDLED = 2;

Using #define:

#define REDLED 2

Variables

Define most of variable such as threshold distance etc. at the top of code under pin/other constants

Example

int minDistance = 10;
int userInput;

void setup()

Explanation

This function is ran once at the beginning.

  • Setting pin modes (explained below)
  • Beginning serial communication (explained below)

Example

void setup() {
    // setting pinModes (explained below)
    // begin serial communication (explained below)
}

pinMode(pin, mode)

Explanation

We use the pinMode(pin, mode) function to tell the Arduino how we want to use the pin. The common ones are OUTPUT and INPUT.

Example

pinMode(REDLED, OUTPUT);

void loop()

Explanation

Contains all the stuff to do forever (reading pins, outputting to pins/serial monitor). This function is ran forever after the void setup() has ran.

Example

void loop() {
    // stuff to do forever
}

delay(time_millieseconds)

Explanation

delay(time_millieseconds) to 'pause' our program for a given time.





Also: delayMicroseconds(time_microsends)

Example

delay(time_millieseconds) example:

delay(200)

delayMicrosends(time_microseconds) example:

delayMicroseconds(200)

digitalWrite(pin, value)

Explanation

digitalWrite(pin, value) to 'write' a value to our pin. The value argument can be either LOW or HIGH.

Example

digitalWrite(REDLED, HIGH);

digitalRead(pin)

Explanation

digitalRead(pin) gives the value currently being read from the specified pin.

Example

digitalRead(BUTTONPIN)

Serial communication

Serial communication is talking between the Arduino and another source, such as the serial monitor.

Explanation

Enabling Serial Communication

Serial.begin(<baud_rate>) in void setup() (example to the right).

Printing to Serial Monitor

  • Serial.println(<message>) (advance)
  • Serial.print(<message>) (no advance)

Reading from Serial Monitor

  • Serial.available() - check if there is input
  • Serial.read() to read a byte (like a character)
  • Serial.parseInt() to read an integer
  • Serial.parseFloat() to read a float

More information on serial communication available here.

To print a string a non-string variable using one .print/.println, you must first convert the variable to a string, shown with the Serial.print in the 'Reading Examples' to the right.

Example

void setup() {
    Serial.begin(9600);
}

Printing example:

Serial.println(buttonState);

Reading examples:

int userInput;

// other code

void loop() {
    if (Serial.available() > 0) {
        userInput = Serial.parseInt()
    }
    
    Serial.print("User entered " + String(userInput));
    
    delay(200);
}

analogRead(pin)

Explanation

Used to read data from an analog component, such as a photoresistor. Returns value between 0 and 1023.

Analog components must be connected to an 'analog in' pin on the Arduino or a digital pin with PWM (pins have ~ to denote this).

Example

analogRead(LDRPIN);

Tones - tone(pin, frequency) and - noTone(pin)

Explanation

tone(pin, frequency, [duration]) - Duration is optional, if not specified it will go forever.

noTone(pin) to stop the tone.

Example

tone(BUZZERPIN, 50);
tone(BUZZERPIN, 50, 500)
noTone(BUZZERPIN)

pulseIn(pin, value)

Explanation

pulseIn(pin, value) to time how long a pulse lasts. Value is for the type of signal to time (HIGH or LOW).

Once a signal of <value> has been detected, times how long (microseconds) it lasts till the opposite of <value> detected.

Example

pulseIn(PIN, HIGH);
